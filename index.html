Adaptive Energy-Aware Scheduling Algorithm
AEAS — Version 2.0 | Novel OS Scheduling Algorithm
CORE INNOVATION

AEAS is a hybrid preemptive scheduling algorithm that combines dynamic priority balancing with per-slice DVFS adaptation. Unlike traditional algorithms that run at fixed CPU frequencies, AEAS computes an urgency score for each process at every scheduling decision point, then maps that score to an appropriate DVFS operating point—minimizing energy without sacrificing responsiveness.

URGENCY SCORE FORMULA
// Urgency ∈ [0, 1]
U(p,t,n) = 0.35·Pnorm + 0.30·Wnorm + 0.20·Bnorm + 0.15·Qnorm
P: priority weight | W: waiting pressure | B: burst length | Q: queue pressure
ENERGY MODEL
E = P(f,V) × Texec
P = α·C·f·V² + Pstatic
Texec = Tburst × (fbase / fselected)
P: power (mW) | f: frequency (GHz) | V: voltage | C: capacitance

PSEUDOCODE
ALGORITHM AEAS(processes[], quantum):
  ready_queue ← []
  age_table ← {}
  t ← 0
  
  WHILE unfinished_processes exist:
    ready_queue ← {p | p.arrival ≤ t AND p.remaining > 0}
    
    IF ready_queue is empty:
      t ← next_arrival_time()          // Sleep to next event
      emit_idle_slot(t)
      CONTINUE
    
    FOR EACH p in ready_queue:
      score(p) ← 0.45·priority(p) + 0.30·age(p) + 0.25·sjf(p)
    
    selected ← argmax(score)
    age[selected] ← 0                  // Reset aging
    
    urgency ← compute_urgency(selected, t, |ready_queue|)
    (freq, volt, power) ← DVFS_select(urgency)
    
    slice ← min(quantum, selected.remaining)
    t_exec ← slice × (f_base / freq)  // Freq-adjusted time
    energy ← power × t_exec / 1000
    
    emit_gantt(selected, t, t+t_exec, freq, energy)
    selected.remaining -= slice
    t += t_exec
    
    IF selected.remaining ≤ 0:
      finalize(selected, t)            // Compute WT, TAT
KEY ADVANTAGES OVER TRADITIONAL ALGORITHMS
✓ DVFS per time-slice — adapts voltage/frequency every scheduling decision
✓ Anti-starvation aging — prevents indefinite preemption of low-priority tasks
✓ Idle-state sleep — CPU enters low-power sleep when queue is empty
✓ Hybrid priority+SJF — balances fairness and efficiency
✓ Queue-aware urgency — considers system load for smarter DVFS selection
